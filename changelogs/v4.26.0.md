## 更新日志

### 新功能

- 新增 Exa 网页搜索提供商，支持 Exa 搜索与页面内容提取工具。
- 登录页新增公开版本信息展示，便于排查 WebUI 静态资源版本与运行中后端版本不一致的问题。

### 优化

- 为知识库上传与插件上传增加更严格的文件名收敛逻辑，降低路径穿越风险。
- OpenAPI Key scopes 新增 `data`，与当前数据类路由权限模型保持一致。
- GitHub Actions 工作流统一升级到 `actions/checkout@v7`。
- 更新默认上下文配置：`max_context_length = -1`、`dequeue_context_length = 1`，并兼容无限上下文语义。

### 修复

- 修复标准 crontab 星期字段在 APScheduler 中的兼容问题，避免 `0/7 = Sunday` 被错误解释。
- 修复 QQ 官方平台发送消息时 `At` 组件被丢失的问题。
- 补充正式版 `4.26.0` 的版本号、文档与 changelog 对齐。

## Changelog

### New Features

- Added Exa as a web search provider, including Exa search and page content extraction tools.
- Added public version details to the login page to help diagnose mismatches between served WebUI assets and the running backend.

### Improvements

- Hardened uploaded filename handling for knowledge base and plugin uploads to reduce path traversal risk.
- Added the `data` OpenAPI key scope to match the current authorization model for data-oriented routes.
- Upgraded GitHub Actions workflows to `actions/checkout@v7`.
- Updated the default context settings to `max_context_length = -1` and `dequeue_context_length = 1`, with compatibility for the unlimited-context behavior.

### Fixes

- Fixed compatibility for standard crontab weekday fields in APScheduler so `0/7 = Sunday` is interpreted correctly.
- Fixed dropped `At` components when sending messages on the QQ Official platform.
- Aligned the final `4.26.0` version number, docs, and changelog metadata.
