## What's Changed

### Features

- Support custom bot IDs during QR-based platform creation (#9067)
- Improve ChatUI attachment display (#9134)
- Sanitize malformed tool call names in `ToolLoopAgentRunner` (#9144)
- Show model metadata in selectors and add a chat token usage indicator (#9161)
- Add event-loop diagnostics and expand diagnostics documentation (#9168)
- Add notes on cross-platform compatibility and Python version support
- Add test coverage for POSIX file URI root preservation (#8906)

### Changes

- Keep model metadata separate from provider definitions (#9161)
- Log event-loop watchdog startup at the info level

### Fixes

- Enable only synced ModelScope MCP servers (#9084)
- Improve MCP startup and shutdown cleanup to avoid idle spinning and leaked contexts (#9070, #9132, #8307)
- Replace fragile duplicate-name matching in the knowledge-base flow with a pre-check (#9121)
- Improve knowledge-base retrieval resilience, cleanup, and CRUD contract alignment (#9122, #9120, #9000)
- Improve web search API key failover
- Enforce ownership when reading ChatUI sessions and secure project update staging (#9141, #9083)
- Improve QQ Official retry behavior when websocket sends return `None` (#8977)
- Preserve QQ Official quoted message context and proactive fallback replies (#9164, #9169)
- Adapt MiMo STT to V2.5 models and reject non-WAV audio payloads (#9118)
- Serialize webchat history timestamps as UTC-aware ISO strings (#9159)
- Sync plugin activation status on enable in the WebUI (#9156)

### Styles

- Polish snackbar styling in the WebUI (#9173)
